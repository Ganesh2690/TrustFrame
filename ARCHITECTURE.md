## 1. Project Overview

**TrustFrame** is a mobile-first evidence capture platform for recording and preserving photo/video
evidence of civil incidents with tamper-evident chain-of-custody. Built as a 14-day POC for
DreamzTech Solutions, targeting B2C events.

| Attribute | Value |
|---|---|
| Client | DreamzTech Solutions |
| Timeline | 14-day POC |
| Target Users | Citizens, field officers |
| Core Value | Frictionless capture + cryptographic tamper evidence |
| Phase 2 (deferred) | AI analysis, blockchain anchoring, deepfake detection, C2PA |

## 2. High-Level Architecture

```
Browser (React SPA)
  └─ LandingPage / CapturePage / TagEventPage / EvidenceReportPage / AdminPage
  └─ SessionContext (localStorage session state)
  └─ api.js (Axios, 60s timeout, /api/* proxy)
         │
         ▼ HTTP
Reverse Proxy (nginx :80 prod / Vite :5173 dev)
  └─ /api/* → FastAPI backend :8000
  └─ /static/* → StaticFiles uploads
         │
         ▼
FastAPI Backend :8000
  ├─ /api/session → sessions.py
  ├─ /api/events  → events.py
  ├─ /api/uploads → uploads.py ──→ storage.py (aiofiles streaming)
  ├─ /api/reports → reports.py      hashing.py (SHA-256)
  ├─ /api/admin   → admin.py        report_tokens.py
  └─ /static/uploads → StaticFiles
         │
         ▼
SQLAlchemy ORM
  ├─ SQLite (dev: ./trustframe_dev.db)
  └─ PostgreSQL 16 (prod: Docker named volume pgdata)

File System
  └─ uploads/images/{uuid}.jpg

```

## 4. Execution Entry Points

| Entry Point | Command | Init Sequence |
|---|---|---|
| Backend server | `uvicorn app.main:app --reload --port 8000` | Load settings → create_all() → makedirs(uploads) → register middleware → mount StaticFiles → register routers |
| Frontend dev | `npm run dev` (from frontend/) | Vite reads .env → starts HMR server :5173 → proxies /api/* and /static/* to localhost:8000 |
| Tests | `pytest tests/ -v` (from backend/) | conftest.py overrides DATABASE_URL → SQLite in-memory engine → StaticPool → creates tables → per-test transaction rollback |
| Alembic | `alembic upgrade head` (from backend/) | Reads alembic.ini (⚠️ hardcoded PostgreSQL URL) → runs DDL migrations |

## 7. End-to-End Workflow Trace

1. **App Open → Session Start**: Browser checks localStorage. If absent → name/phone form → `POST /api/session/start` → UserSession row → session_id stored in localStorage.

2. **QR Entry (alternate)**: QR code encodes `APP_URL?event_id=<code>`. LandingPage reads param → passes presetEventId to CapturePage.

3. **GPS Acquisition**: `navigator.geolocation.getCurrentPosition({enableHighAccuracy:true, timeout:10000})`. Failure → null GPS (capture still allowed). **GPS is client-reported; not server-verified.**

4. **File Selection and Validation**: Two inputs: live camera (`capture=environment`) and gallery. Frontend validates MIME type (image/* or video/*) and file size (≤ MAX_UPLOAD_SIZE_MB × 1024²).

5. **Upload and Provenance Recording**:
   - `POST /api/uploads` (multipart: file + session_id + GPS + client_timestamp + device_info)
   - Backend: validate session → validate MIME → `LocalStorage.save()` (stream + SHA-256) → `compute_session_integrity_hash()` → INSERT MediaAsset → INSERT CustodyEvent(UPLOAD) → INSERT EvidenceReport
   - Returns: `{asset_id, sha256_hash, report_token, report_url}`

6. **Event Tagging (optional)**: `PATCH /api/uploads/{id}/event {event_code}` → sets event_channel_id FK → INSERT CustodyEvent(EVENT_TAGGED).

7. **Report Viewing**: `GET /api/reports/{token}` → token lookup → view_count++ → INSERT CustodyEvent(REPORT_VIEWED) → return ReportResponse (REPORT_VIEWED excluded from public custody_chain).

8. **Admin Monitoring**: `GET /api/admin/stats|uploads|events|reports` with `X-Admin-Secret` header.

## 8. Subsystems and Interface Contracts

| Subsystem | HTTP Surface | Auth | Key Behavior |
|---|---|---|---|
| Session Management | `POST /api/session/start` | None | Creates anonymous UserSession; returns UUID |
| Event Management | `GET/POST /api/events`, `/events/{code}`, `/events/{code}/qrcode` | session_id in POST body | 8-char cryptographically random event codes; QR PNG returned as bytes |
| Upload & Provenance | `POST /api/uploads`, `PATCH /api/uploads/{id}/event` | session_id in form | Core pipeline; atomic create MediaAsset + CustodyEvent + EvidenceReport |
| Evidence Reports | `GET /api/reports/{token}` | Token in URL (bearer) | Token lookup; side-effects: view_count++, REPORT_VIEWED event |
| Admin API | `GET /api/admin/*` | X-Admin-Secret header | Read-only; no pagination; full data dumps |
| File Storage | Internal (StaticFiles for reads) | None | Streams to images/ or videos/; returns relative path |
| Custody Ledger | Internal | None | Append-only by convention; 4 event types; filtered on read |

## 9. State Model

### UserSession States
`Anonymous` → (POST /api/session/start) → `Active` → (clearSession) → `Anonymous`  
Session persists across page reloads via localStorage. No expiry.

### MediaAsset Lifecycle
`Uploading` → `Persisted` → (optional) `Tagged` → `Reported`  
Asset metadata is immutable after creation (no UPDATE endpoints). Event association is the only post-creation mutation.

### CustodyEvent Types (append-only)
1. `UPLOAD` — written at upload time
2. `REPORT_GENERATED` — written at upload time (alongside UPLOAD)
3. `EVENT_TAGGED` — written when PATCH /api/uploads/{id}/event called
4. `REPORT_VIEWED` — written on every GET /api/reports/{token}; **excluded from public chain**

## 11. Data Flow and Persistence

### Database Tables

| Table | Primary Key | Key Fields | Relationships |
|---|---|---|---|
| `user_sessions` | UUID | display_name, phone_number, user_agent, timestamps | → media_assets (1:N), → event_channels (1:N) |
| `event_channels` | UUID | event_code (8-char, UNIQUE, INDEX), name, GPS fields | ← user_sessions, → media_assets (1:N) |
| `media_assets` | UUID | storage_path (UNIQUE), sha256_hash, server_timestamp_utc, GPS fields, device_info, session_integrity_hash, file_size_bytes, mime_type | ← user_sessions, ← event_channels, → custody_events (1:N), → evidence_reports (1:1) |
| `custody_events` | UUID | media_asset_id (FK, INDEX), event_type, timestamp_utc, metadata (JSON) | ← media_assets |
| `evidence_reports` | UUID | media_asset_id (FK, UNIQUE), report_url_token (UNIQUE, INDEX), view_count | ← media_assets (1:1) |

### File System Layout
```
uploads/
  images/    ← MIME starts with image/  (JPEG, PNG, HEIC, etc.)
  videos/    ← MIME starts with video/  (MP4, MOV, etc.)
```
Served via `GET /static/uploads/{path}`. Never automatically deleted.

## 12. Security and Access Boundaries

### Trust Levels

| Principal | How Identified | What They Can Do |
|---|---|---|
| Anonymous User | None | Create sessions; view public QR codes |
| Session Bearer | session_id UUID in form | Upload files; create/list events |
| Report Token Bearer | 256-bit URL token | View single evidence report + metadata |
| Admin | X-Admin-Secret header | Read all data (no write operations) |

### Key Security Findings

| OWASP Category | Finding | Severity | Status |
|---|---|---|---|
| A01 Broken Access Control | Admin uses plain string compare (not hmac.compare_digest) | MEDIUM | Not mitigated |
| A01 Broken Access Control | session_id is client-supplied (no ownership proof) | LOW | By design (POC) |
| A04 Insecure Design | GPS, device info, client_timestamp are entirely client-reported | MEDIUM | Documented in Scope.txt |
| A04 Insecure Design | File type validated from Content-Type header (not magic bytes) | MEDIUM | Not mitigated |
| A05 Misconfiguration | SECRET_KEY and REPORT_TOKEN_SECRET loaded but unused | LOW | Dead config |
| A08 Data Integrity | sha256_hash not re-verified when file is served | LOW | Not implemented |
| A09 Security Logging | No structured logging or security event log | MEDIUM | Not implemented |

## 16. End-to-End Operational Runbook

### RB-01: Capture Evidence
1. Open app / scan QR code → create session
2. Allow GPS → select file (camera or gallery)
3. Wait for upload → tag event (optional)
4. Share report URL from EvidenceReportPage

### RB-02: Verify File Integrity
```bash
sha256sum downloaded_file.jpg
# Windows: Get-FileHash downloaded_file.jpg -Algorithm SHA256
# Compare with hash shown on evidence report
```

### RB-03: Admin Monitoring
```bash
curl -H "X-Admin-Secret: <secret>" http://<HOST>/api/admin/stats
curl -H "X-Admin-Secret: <secret>" http://<HOST>/api/admin/uploads
```

### RB-04: Troubleshoot Uploads

| Symptom | Likely Cause | Resolution |
|---|---|---|
| 422 | Missing form fields | Check request format |
| 413 | File too large | Increase MAX_UPLOAD_SIZE_MB |
| 404 on session | Invalid session_id | Re-run POST /api/session/start |
| 500 | DB/disk failure | Check `docker compose logs -f backend` |

### RB-05: DB Maintenance
```bash
docker compose exec db pg_dump -U trustframe trustframe_db > backup_$(date +%Y%m%d).sql
docker compose exec backend alembic upgrade head
docker compose exec db psql -U trustframe -d trustframe_db
```

## 18. Executive Summary for New Joiners

### What Is It?
A mobile-first evidence capture platform where any person can record and preserve photo/video evidence
with tamper-evident chain-of-custody. No account required — frictionless by design.

### The Three Most Important Files
1. `backend/app/routers/uploads.py` — Core pipeline (everything: file save, hash, custody, report)
2. `backend/app/models/` — Complete data model
3. `frontend/src/pages/CapturePage.jsx` — User-facing heart of the product

### The Three Most Critical Gaps
1. **No CI/CD** — All deploys are manual with no automated verification gate
2. **No transactional upload** — File on disk but DB fails → inconsistent state with no cleanup
3. **No rate limiting** — Upload endpoint is unguarded against abuse

### How to Contribute
1. Clone + run backend (`uvicorn app.main:app --reload`) and frontend (`npm run dev`)
2. Run `pytest tests/ -v` — all 20 tests should pass
3. For API changes: update Pydantic schemas, check `http://localhost:8000/docs`
4. For DB changes: create Alembic migration (`alembic revision --autogenerate -m "description"`)
5. See Section 17 for prioritized improvements backlog
